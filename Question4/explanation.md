Question 4: Real-Time Log Monitoring
Objective

The objective is to continuously monitor a log file, process newly added entries in real time, extract ERROR messages, save them in a separate report, and suppress unnecessary error output.

Command Pipeline

The main command pipeline is:

tail -f server.log | grep --line-buffered "ERROR" > error_report.txt 2>/dev/null
tail -f

The tail -f command displays the end of a log file and continuously waits for new lines to be added.

This allows the log file to be monitored in real time without repeatedly opening and checking the file manually.

Pipe |

The pipe operator connects the output of one command to the input of another command.

In this pipeline, the output generated by tail -f is passed directly to grep.

This avoids the need to create an intermediate file.

grep

The grep command searches for specific text patterns.

In this solution, it filters the log entries and selects only lines containing:

ERROR

The --line-buffered option ensures that matching lines are processed immediately, which is important for real-time monitoring.

>

The > redirection operator redirects the output of grep to:

error_report.txt

This creates a separate report containing only the extracted error messages.

2>/dev/null

The 2> operator redirects standard error output.

The special file /dev/null discards anything sent to it.

Therefore:

2>/dev/null

suppresses unnecessary error messages from appearing on the terminal.

Result

The monitoring pipeline successfully detected newly added log entries in real time and extracted only the ERROR messages.

The INFO messages were ignored by grep, and the error messages were stored in the separate error_report.txt file.

Efficiency

The pipe allows commands to work together without intermediate files.

tail -f continuously monitors the log, grep filters only relevant messages, redirection stores the results, and /dev/null suppresses unnecessary error output.
